这是我早期的项目，一个学生信息查询系统（好吧其实是书本例题）
当时我为了提高我的工作效率（我是班委），几乎完全照抄书本
后来感觉书本写的太繁琐（太烂），自己一边向Deep seek学习一边修改这一塌糊涂的代码（主要是书本写的又长又臭，仿佛为了降低AI率魔改了一番）
后面实际体验并不好，我就把他菜单改了（什么功能都要按一次1234选择，什么查询功能按一次，按人名查按一次），
保留增删查改四个方向，统一输入窗口，并且尝试把“按完整名字查询”等改成“检索符合片段”，
并且达成在一个窗口可以输入中文、英文、数字，可以检索序号、名字、地址、手机号码和身份证号码（虽然是要输入#@辨别序号还是名字，是伪搜索框
但这的确提高了我的工作效率
并且，只要不保存，就不会污染txt文件里的数据
后面，我又加入了log系统，但是受限于个人实力以及实际作用不大，我并没有很好地研究这部分功能
最后，我尝试把log系统和structFunc（增删查改）分离，main函数变成纯粹的菜单函数，分出去的文件方便后面修改升级（可能？）
分文件只是一次尝试，因为一开始没有这个想法，后面一时兴起搞的可能分得不完全，或者没必要分，就当我的text啦

以及，我并没有把真正的classdata文件上传，里面只要张三李四，格式方面按照序号/名字/地址/手机/身份证在elsx里拉表格，然后全部复制到classdata就行
（我就是用这个方式导入数据的）
但log文件是真真正正从构建log开始所有的测试数据，就当给我自己一个纪念

点击exe,按照菜单按数字就行，
